1. Overview

  • Framework: FastAPI on Python, served with Uvicorn (ASGI).
  • API prefix: all routes live under /api. Protected routes use JWT Bearer tokens.
  • Database: SQLAlchemy 2.x; default sqlite:///./trainmate.db — override with DATABASE_URL.
  • AI: Groq for session reports and coach chat; OpenRouter (via openai client) as optional provider.
  • Email: verification and password-reset codes via SMTP or Resend.
  • Optional ML: server-side Keras classification at /api/ml/classify when model files exist under ai/models/ or ML_MODELS_DIR.

Run locally

.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

Health check: GET http://127.0.0.1:8000/api/health

8. Backend data management

8.1 Schema creation & SQLite upgrades

  • On startup: Base.metadata.create_all(bind=engine) ensures tables exist.
  • _ensure_sqlite_columns() adds missing columns on SQLite for older DB files (lightweight migrations without Alembic).

8.2 Tables

  • users — identity, password hash, verification & reset code hashes.
  • user_profiles — anthropometrics, notes, profile image (base64).
  • user_plans — plan category, duration, before/after photos, onboarding flag.
  • workout_sessions — per-session exercise metrics and optional AI session_report.

8.3 Request-scoped DB sessions

get_db() yields one SQLAlchemy session per request and closes it in a finally block — standard FastAPI pattern.

8.4 Environment variables (see app/config.py)

  • JWT_SECRET_KEY, JWT_ALGORITHM, ACCESS_TOKEN_EXPIRE_MINUTES
  • DATABASE_URL
  • GROQ_API_KEY, GROQ_MODEL, LLM_PROVIDER, OPENROUTER_*
  • SMTP_* or RESEND_API_KEY
  • EXPOSE_VERIFICATION_CODES (dev convenience)
  • ML_MODELS_DIR (optional path to Keras artifacts)

10. Security & behavior notes

  • Passwords hashed with bcrypt; email codes stored as HMAC digests. Password reset codes are never returned in JSON responses.
  • LLM_PROVIDER=auto prefers OpenRouter when a key exists; otherwise Groq.
  • Chat builds rich workout statistics context to ground answers and reduce hallucinated personal facts.
  • Workout kcal uses a MET-style heuristic from exercise name plus weight/duration when available.
